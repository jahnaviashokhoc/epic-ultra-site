Framework

Security & Compliance

The EPICUltra framework enforces security end-to-end — from data access and agent execution to audit trails and kill-switch controls. Every layer is built for regulated enterprise environments where accountability is non-negotiable.

Access & Identity

Control Who Sees What, Always

Identity-propagated access, row-level security, and role-based controls enforced at every layer — no shared elevated tokens, no exceptions.

🔑
RBAC — Role-Based Access Control
  • EPIC Ultra enforces data-level security end-to-end through the Policy/Entitlements Service (PES), which resolves effective access as the intersection of agent-declared dataset scopes and user entitlements.
  • All business-data queries execute on behalf of the requesting user via identity propagation — shared elevated tokens are prohibited.
  • Unity Catalog (Databricks) provides baseline catalog/schema/table privileges, with row-level and column-level security enforced via row filters and column masks.
📋
Audit Logging
  • Every agent execution produces auditable records capturing agent_id, agent_version, user_id, datasets touched, tool calls performed, and LLM token usage.
  • All operations — authentication events, dataset access, ingestion runs, reference table changes, and tool invocations — are correlated by a request-scoped correlation_id/trace_id propagated across all services.
  • Audit entries are retained per compliance policy and accessible only to authorized roles.
🔄
Agent Versioning
  • Each agent is independently versioned and stored with its configuration in PostgreSQL.
  • When a new agent version is deployed, routing automatically points to the new version while the old version is sunset gracefully.
  • The agents.md manifest declares each agent's allowed datasets, tool allowlists, and version metadata, ensuring traceability and rollback capability.
🏗️
Environment Segregation
  • Agent runtimes are deployed into private network segments with no public IP and egress-deny-by-default policies, with only allowlisted internal endpoints reachable.
  • Each agent runs in an independent container with isolated resource limits, health checks, and network policies enforcing least-privilege communication.
  • All denied outbound attempts are logged with correlation identifiers for security observability.
🔒
Secure Microservices
  • EPIC Ultra employs a microservices architecture with clear separation of concerns across orchestration, AI processing, and UI layers, with TLS encryption enforced for all inter-service communication.
  • Agents access business data exclusively through the Query Execution Service (QES) — never directly to Databricks — with server-side query validation.
  • Secrets are stored only in an approved secret manager, and telemetry avoids sensitive payload capture using hashed identifiers.
🔍
Data Lineage
  • EPIC Ultra tracks data from ingestion to AI consumption, capturing source system, load timestamps, job_id, and target Unity Catalog identifiers for all data ingested from external sources such as ETRM systems.
  • Ingested tables are registered in Unity Catalog with classification tags and ownership for downstream policy enforcement.
  • At the agent layer, every execution logs the specific datasets accessed, queries executed via QES, and tool calls performed — providing end-to-end visibility into how external data flows through the platform and is utilized by AI agents.
AI Transparency

Explainability & Runtime Control

Every AI decision is traceable, every agent is measurable, and every deployment has an off switch.

🧠
Reasoning / Chain of Thought
  • EPIC Ultra AI agents capture structured rationale and reasoning steps for each decision made during workflow execution, providing transparency into how the agent arrived at its conclusions.
  • This chain of thought includes references to the datasets, rules, and tool invocations used in the decision-making process.
  • These explanations are stored in a security-safe manner suitable for audit and compliance review, without persisting sensitive raw prompts or responses.
📊
Agent Evaluation
  • EPIC Ultra tracks agent performance metrics over time including execution latency, success/failure rates, LLM token consumption, and step counts.
  • These metrics enable comparison of agent versions to measure improvements in accuracy, efficiency, and reliability as agents evolve.
  • This continuous evaluation framework supports data-driven decisions on when to promote new agent versions or roll back underperforming ones.
⚡
Kill Switch Trigger
  • EPIC Ultra provides a kill switch mechanism that allows administrators to immediately halt a running agent or disable an agent version across the platform, preventing further execution in case of anomalous behavior, security concerns, or erroneous outputs.
  • The kill switch leverages the agent container strategy to instantly reroute or terminate agent workloads with zero impact to the core system or other agents.
  • All kill switch activations are logged with the triggering user, reason, and timestamp for audit and compliance purposes.
Core Services

The Control-Plane Services

PES and QES are the security backbone of the platform — every authorization decision and every data access flows through them.

Policy & Entitlements Service
PES
  • PES is a security-critical control-plane service that manages and enforces dataset and tool access policies across the platform.
  • It resolves effective permissions for each request by computing the intersection of the agent's declared scope and the user's entitlements.
  • Every authorization decision — granted or denied — is logged with correlation identifiers for audit and compliance.
Query Execution Service
QES
  • QES is the sole gateway for all business data access, executing Databricks queries and RAG retrieval on behalf of the requesting user using identity propagation.
  • It validates and constrains all queries to read-only statements against allowlisted tables, enforces row limits, and blocks risky constructs such as DDL/DML operations.
  • This ensures that neither agents nor services can bypass data-level security or directly access Databricks with elevated credentials.

Built for Regulated Environments

No workarounds, no shared credentials, no black-box execution. Every action governed and traced from end to end.
